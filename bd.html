Задание 3. Работа с базой данных MySQL

В рамках третьей лабораторной работы была создана база данных MySQL для хранения анкет пользователей. Ниже представлена структура базы данных, скриншоты процесса создания таблиц и пояснения к каждому этапу.

Структура базы данных

База данных спроектирована в третьей нормальной форме . Используются три таблицы с отношением "один ко многим".

applications

Основная таблица с анкетами

id (PK), full_name, phone, email, birth_date, gender, biography, contract_accepted, created_at
← 1 : N →

application_languages

Таблица связи

id (PK), application_id (FK), language_id (FK)
← N : 1 →

programming_languages

Справочник языков

id (PK), name

Таблица applications (анкеты)

Поле Тип Описание
id INT UNSIGNED Первичный ключ, автоинкремент
full_name VARCHAR(150) ФИО (обязательное)
phone VARCHAR(20) Телефон
email VARCHAR(100) Email
birth_date DATE Дата рождения
gender ENUM('male','female') Пол
biography TEXT Биография (необязательное)
contract_accepted TINYINT(1) Согласие с контрактом
created_at TIMESTAMP Дата создания записи

Таблица programming_languages (справочник языков)

Поле Тип Описание
id INT Первичный ключ, автоинкремент
name VARCHAR(50) Название языка программирования (уникальное)

Таблица application_languages (связь анкет и языков)

Поле Тип Описание
id INT UNSIGNED Первичный ключ, автоинкремент
application_id INT UNSIGNED Внешний ключ → applications(id) ON DELETE CASCADE
language_id INT Внешний ключ → programming_languages(id)

2. Создание таблицы programming_languages

Что происходит: Создаётся справочная таблица языков программирования. В неё вставляются 12 допустимых языков: Pascal, C, C++, JavaScript, PHP, Python, Java, Haskell, Clojure, Prolog, Scala, Go.

CREATE TABLE IF NOT EXISTS programming_languages (
   id INT AUTO_INCREMENT PRIMARY KEY,
   name VARCHAR(50) NOT NULL UNIQUE
);

3. Создание таблиц applications и application_languages

Что происходит: Создаётся основная таблица applications для хранения анкет, а также таблица связи application_languages для реализации отношения "один ко многим" между анкетами и языками программирования.

-- Основная таблица анкет
CREATE TABLE IF NOT EXISTS applications (
   id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
   full_name VARCHAR(150) NOT NULL,
   phone VARCHAR(20) NOT NULL,
   email VARCHAR(100) NOT NULL,
   birth_date DATE NOT NULL,
   gender ENUM('male', 'female') NOT NULL,
   biography TEXT,
   contract_accepted TINYINT(1) NOT NULL DEFAULT 0,
   created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Таблица связи (один ко многим)
CREATE TABLE IF NOT EXISTS application_languages (
   id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
   application_id INT UNSIGNED NOT NULL,
   language_id INT NOT NULL,
   FOREIGN KEY (application_id) REFERENCES applications(id) ON DELETE CASCADE,
   FOREIGN KEY (language_id) REFERENCES programming_languages(id)
);

Схема связей таблиц

Примечание: Использование внешних ключей гарантирует целостность данных. При удалении анкеты (ON DELETE CASCADE) автоматически удаляются все связанные с ней записи о языках программирования.
